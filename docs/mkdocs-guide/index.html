Гайд по MkDocs

  1. Поставить на комп Python если не стоит ещё.
  2. Через его консоль поставить mkdocs. pip install mkdocs
  3. Всё управление идёт через консоль.
  4. Новый проект (проект создаётся там, где запущена консоль). mkdocs new my-project cd my-project
  5. Запуск живого превью (будет доступно по адресу http://127.0.0.1:8000/) mkdocs serve
  6. Билдинг проекта в статичный сайт: mkdocs build
  7. Вся работа с текстом идёт в файлах .md. Синтаксис markdown.
  8. В файле mkdocs.yml все настройки проекта и его структура.
  9. Я изменил дефолтные настройки на свои чтобы сайт билдился в папку docs которая используется в github pages. А рабочие файлы лежат в папке mkdocs.
  10. После того как я отдам репозиторий, работа по изменению доков будет выглядеть так:
  11. Правки вносятся в файлы в папке mkdocs.
  12. Картинки добавляются в папку mkdocs/img.
  13. Робот у себя запускает билд сайта (см. выше).
  14. Робот делает пуш в мастер гита.
  15. Какое-то время это могу делать я.
